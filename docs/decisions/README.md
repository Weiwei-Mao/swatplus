# Decision Records

Create a decision record when choosing a code change, build/configuration policy, knowledge-structure change, or other approach that future work may question.

Decision files use `YYYY-MM-DD-short-decision.md` and explain context, evidence, considered alternatives, the selected decision, consequences, and verification. Do not use a decision record as a substitute for canonical model knowledge.

Use the [decision template](../templates/decision-record.md).

## Decision catalog

The initial documentation and knowledge-base designs are stored under [`../superpowers/specs/`](../superpowers/specs/). Future operational or model-change decisions will be listed here.
