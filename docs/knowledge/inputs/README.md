# Inputs

Knowledge about SWAT+ input files, their readers, parsed fields, defaults, validation, and destination data structures belongs here.

A strong input note answers:

1. Which file and record define the value?
2. Which routine opens and reads it?
3. Which module/type/field stores it?
4. What defaults, limits, units, and object indices apply?
5. Which controls or processes consume it?

Use an end-to-end record in [`../../traces/`](../../traces/README.md) when proving a specific input-to-output path.
