# Simulation Objects

Object-level behavior belongs here: full HRUs, HRU-LTE, routing units, aquifers, channels, reservoirs, wetlands, recall, exco/delivery, and other object types dispatched by SWAT+.

Each object note should explain:

- How the object is declared and connected by inputs.
- Its identifying indices and primary data types.
- Its control routine and internal process order.
- Inflows, state, outflows, and outputs.
- Which alternate implementations can be confused with it.

Cross-object movement should also link to a trace under [`../../traces/`](../../traces/README.md).
